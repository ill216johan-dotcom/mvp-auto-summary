## Jitsi + Jibri (добавлено 02.03.2026)

| Компонент | Расположение | Примечание |
|-----------|-------------|------------|
| docker-jitsi-meet | `/opt/jitsi-meet/` | Jitsi + Jicofo + JVB + Prosody + Jibri |
| Jitsi config | `/opt/jitsi-meet/.env` | PUBLIC_URL=https://ff-meet.duckdns.org |
| Jibri compose | `jibri.yml` + `jibri-override.yml` | override монтирует /mnt/recordings |
| finalize.sh | `/opt/jibri/finalize.sh` | Переименование `LEAD-{ID}-conf` → `{ID}_YYYY-MM-DD.mp4` |
| Recordings | `/mnt/recordings/` | Смонтировано в n8n как `/recordings:ro` |
| **Host nginx** | `/etc/nginx/sites-available/` | Reverse proxy на порту 443 для обоих доменов |
| **SSL сертификаты** | `/etc/nginx/ssl/` | Let's Encrypt через acme.sh |

**Как создать комнату для записи:**
1. Открыть `https://ff-meet.duckdns.org/LEAD-{ID}-conf` (например LEAD-4405-conf)
2. Нажать кнопку **"Запись"** в меню (три точки)
3. Jibri автоматически запишет и сохранит в `/mnt/recordings/YYYY/MM/DD/{ID}_YYYY-MM-DD_HH-MM.mp4`
4. WF01 подберёт файл в течение 5 минут и запустит транскрипцию

**Запустить Jitsi/Jibri после рестарта:**
```bash
cd /opt/jitsi-meet
docker compose -f docker-compose.yml -f jibri.yml -f jibri-override.yml up -d
systemctl start nginx  # reverse proxy для HTTPS
```

**Продление SSL сертификатов** (автоматически через cron):
```bash
~/.acme.sh/acme.sh --renew -d ff-meet.duckdns.org --ecc --reloadcmd "systemctl reload nginx"
~/.acme.sh/acme.sh --renew -d dify-ff.duckdns.org --ecc --reloadcmd "systemctl reload nginx"
```

### Миграция на постоянный домен

Текущая конфигурация использует бесплатный DuckDNS поддомен (`ff-meet.duckdns.org`). Для продакшена рекомендуется миграция на постоянный домен:

**Шаги миграции:**
1. Купить домен (например `meet.ff-platform.ru`)
2. Создать A-запись: `meet.ff-platform.ru → 84.252.100.93`
3. Получить SSL сертификат:
   ```bash
   ~/.acme.sh/acme.sh --issue -d meet.ff-platform.ru --standalone
   ```
4. Установить сертификат:
   ```bash
   ~/.acme.sh/acme.sh --install-cert -d meet.ff-platform.ru \
     --key-file /etc/nginx/ssl/meet.ff-platform.key \
     --fullchain-file /etc/nginx/ssl/meet.ff-platform.crt
   ```
5. Обновить nginx config (`/etc/nginx/sites-available/ff-meet.conf`):
   - Заменить `ff-meet.duckdns.org` на новый домен
   - Обновить пути к сертификатам
6. Обновить Jitsi PUBLIC_URL:
   ```bash
   sed -i 's|PUBLIC_URL=.*|PUBLIC_URL=https://meet.ff-platform.ru|' /opt/jitsi-meet/.env
   cd /opt/jitsi-meet && docker compose up -d --force-recreate web
   ```
7. Перечитать nginx: `systemctl reload nginx`

**Если есть существующий Jitsi сервер** (например `global-meet.ff-platform.ru`):
- Вариант А: Перенести Jibri на существующий сервер + настроить NFS
- Вариант Б: Использовать существующий домен для текущего сервера (как описано выше)
- Вариант В: Оставить текущий сервер как тестовый, продакшн на существующем
