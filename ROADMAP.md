- Файлы записей хранятся в Bitrix Disk (`RECORD_FILE_ID`)

- ✅ `transcribe_pending_calls()` — скачивает через Disk API, fallback на прямой URL
- ✅ Webhook сервер (`app/webhook_server.py`) на порту 8009 — новые звонки в реальном времени
- ✅ Scheduler job `poll_new_recordings` — каждые 30 мин проверяет новые записи

## Архитектура потока данных

```
Bitrix24 CRM
  ├── crm.activity.list       → звонки, email, комменты → PostgreSQL
  ├── voximplant.statistic.get → RECORD_FILE_ID         → bitrix_calls.record_file_id
  └── disk.file.get(file_id)  → DOWNLOAD_URL → MP3      → Whisper → transcript_text

Whisper (faster-whisper, CPU)
  └── /v1/audio/transcriptions → текст → bitrix_calls.transcript_text

Claude 3.5 Haiku (LLM)
  └── generate_bitrix_summaries() → саммари за каждый день → bitrix_summaries

Dify RAG
  └── dataset per client (ФФ-XXXX) → документы = саммари по дням

Webhook (порт 8009)
  └── OnVoximplantCallEnd → мгновенная транскрибация нового звонка

Scheduler (APScheduler)
  ├── 22:00 — полная синхронизация Bitrix + транскрибация + саммари
  ├── */30m — poll_new_recordings (новые записи за 25 ч)
  └── */5m  — check_pending (Whisper очередь)
```

## Ключевые технические решения

### Записи звонков
- **Источник:** Bitrix Disk API (`disk.file.get`) — не MegaPBX (недоступен)
- **Период:** только звонки после ~июня 2025 имеют `RECORD_FILE_ID`
- **Matching:** voximplant → bitrix_call по lead_id + дата ±5 мин

### Идентификаторы клиентов
- `diffy_lead_id` = активный договор (`ФФ-4405`) или `BX-LEAD-NNNNN`
- `contract_number` = все договоры (`ФФ-4405 / ФФ-2577`)
- Датасет Dify именуется по всем договорам

### Идемпотентность
- Все INSERT используют `ON CONFLICT DO NOTHING / DO UPDATE`
- `transcript_status` переходы: `no_record` → `pending` → `done/failed`
- `failed` можно сбросить в `pending` и перезапустить
- Саммари пишутся с `ON CONFLICT DO NOTHING` — повтор не создаёт дубликат

## Известные ограничения

| Ограничение | Детали |
|-------------|--------|
| Записи до июня 2025 | Файлы удалены из хранилища Bitrix — только метаданные |
| event.bind (webhook) | Требует OAuth, не работает с webhook-токеном — используем polling |
| Whisper CPU | ~1.3x реального времени, для 10 мин звонка ~13 мин транскрибации |
| Batch enrichment | bitrix_call_id пустой у старых записей — matching по lead+дата |
