# MVP Auto-Summary — Статус на 26.02.2026 (v2 — MVP Complete)

## Архитектура (после миграции на Dify + per-client RAG)

```
Jitsi/Jibri → /recordings/*.webm
                     ↓
           [WF01 — каждые 5 мин]
                     ↓
        transcribe (SpeechKit STT)
                     ↓
              [PostgreSQL]
           processed_files
                     ↓
        ┌────────────┴────────────┐
        ▼                         ▼
[WF03 — 22:00]             [WF02 — 23:00]
   per-client                 daily digest
 GLM-4 summary              GLM-4 summary
        ↓                         ↓
  Dify KB (LEAD-XXXX)      Telegram (ссылки)
  + .md на nginx           + ссылки на Dify/md
        ↓
  Куратор: Dify Chatbot
  Вопрос: "ФФ-4405 что обсуждали?"
  → RAG по всей истории клиента
```

## Следующие шаги для деплоя

### 1. Применить миграцию БД
```bash
docker exec mvp-autosummary-postgres-1 \
  psql -U n8n -d n8n -f /scripts/migrate_db_v2.sql
```

### 2. Пересобрать и запустить (добавлен summaries-nginx)
```bash
cd /opt/mvp-autosummary
git pull
docker compose up -d --build
```

### 3. Создать датасеты в Dify
```bash
python3 scripts/setup_dify_datasets.py --api-key YOUR_KEY ...
```

### 4. Создать Chatbot в Dify UI
- Открыть http://84.252.100.93:8080
- Studio → New App → Chatbot
- Название: «ФФ Ассистент Куратора»
- Подключить ВСЕ KB (общий + все LEAD-*)
- Вставить системный промпт из `scripts/setup_dify_chatbot.py`
- Нажать Publish

### 5. Обновить воркфлоу в n8n
```bash
# Импортировать через n8n UI или API:
# Settings → Import from file → выбрать файлы из n8n-workflows/
```

### 6. Активировать WF06 (опционально)
```bash
# В n8n UI: найти WF06 → включить toggle Active
```

### 7. Загрузить оферту в общий датасет
- Открыть Dify → Knowledge Base → «Общая документация ФФ Платформы»
- Add document → загрузить PDF или TXT файлы
