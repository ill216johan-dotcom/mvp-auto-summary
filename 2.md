 Диагностика

 Возможные причины (требуется проверка логов):

1. Rate limits Bitrix API
   
   - Синхронизация падает после ~N запросов
   - Ошибка 429 Too Many Requests
   - Решение: добавить exponential backoff (уже есть в коде?)

2. Ошибки при создании датасетов
   
   - Dify возвращает ошибку 500
   - Пропускаются лиды без контракта
   - Решение: логировать ошибки, retry

3. Фильтрация по статусам лидов
   
   - Синхронизируются только статусы 1-7
   - ~40% лидов отфильтровываются как NEW/JUNK/closed
   - Это ПРАВИЛЬНО по бизнес-логике

4. Недостаточно активности
   
   - Датасет создается только при ПЕРВОМ саммари
   - Лиды без звонков/писем/комментов не создают датасет
   - Это ПРАВИЛЬНО но нужно проверить
   
   Требуемая проверка
   -- На сервере: проверить сколько лидов synced
   SELECT
    bitrix_entity_type,
    COUNT(*) as total,
    COUNT(dify_dataset_id) as with_dataset,

   FROM bitrix_leads
   GROUP BY bitrix_entity_type;
   -- Проверить сколько саммари создано

   SELECT
    summary_date,

   FROM bitrix_summaries

   def get_bitrix_dataset_map(self) -> dict[str, str]:

    with self.cursor() as cur:

            """
            SELECT diffy_lead_id, dify_dataset_id
            FROM bitrix_leads
            WHERE dify_dataset_id IS NOT NULL
            """

        return {row[0]: row[1] for row in cur.fetchall()}
   
   def save_bitrix_dataset_mapping(self, diffy_lead_id: str, dataset_id: str) -> None:
