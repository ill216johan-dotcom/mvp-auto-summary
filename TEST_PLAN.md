## 🧪 Этапы тестирования

### Шаг 1: Подготовка
- [x] Очистить в БД старые саммари для этих 4 клиентов
- [x] Сбросить `dify_dataset_id` (чтобы проверить создание с нуля)
- [x] Выполнить batch_enrich_recordings для этих 4 ID (подтянуть RECORD_FILE_ID из voximplant)

### Шаг 2: Выгрузка и Транскрибация
Запускаем скрипт, который изолированно отработает только этих 4 клиентов:
1. Выгрузит `crm.activity` (звонки, письма, комментарии)
2. Запустит транскрибацию `pending` звонков
3. Убедиться, что скрипт не падает из-за долгой транскрибации (таймаут Whisper = 15 минут)

**Ожидания:**
- Звонки до июня 2025 года: нет записей (это норма, Битрикс их удалил), останутся только метаданные (длительность, номер).
- Звонки после июня 2025 года: скачиваются через `disk.file.get` и транскрибируются в текст.

### Шаг 3: Генерация саммари
1. Выполняется вызов Claude для генерации markdown-саммари по каждому дню активности.
2. Проверяем качество промпта:
   - Нет ли галлюцинаций (Claude не должен выдумывать текст разговора, если транскрипта нет)
   - Если транскрипт есть — он должен быть адекватно суммаризован в блоке "Звонки"
   - Письма (для ФФ-4405) должны быть отображены
3. Проверяем устойчивость:
   - Если LLM отвалится по rate limit, скрипт должен продолжить работу и позволить перезапуск.

### Шаг 4: Интеграция с Dify
1. Проверяем, что создано 4 новых датасета в Dify.
2. Датасет для первого контакта должен называться `ФФ-4405 / ФФ-2577`.
3. Все сгенерированные markdown-файлы должны быть загружены как документы в соответствующие датасеты.

### Шаг 5: Проверка RAG
Идём в интерфейс Dify Chatbot и задаём вопросы:
1. *"Расскажи историю общения с ФФ-2511"* (должен выдать полную сводку).
2. *"О чём договорились с ФФ-4405 в последнем разговоре?"* (проверка транскриптов).
3. *"Что было у ФФ-2577?"* (должен найти датасет `ФФ-4405 / ФФ-2577` и ответить).
