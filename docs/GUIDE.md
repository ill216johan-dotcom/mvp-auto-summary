## 🤔 Что вообще делает эта система?

**Коротко:** Куратор провёл созвон → система сама записала разговор → расшифровала текст → сделала краткий отчёт → отправила в Telegram.

**Подробнее по шагам:**

1. Куратор звонит, запись сохраняется в папку на сервере
2. Система замечает новый файл и расшифровывает его через Яндекс SpeechKit (это как Google Translate, только для голоса в текст)
3. Текст расшифровки сохраняется в базу данных
4. Каждый день в 23:00 система собирает все расшифровки за день и делает общий дайджест
5. Если написать `/report` в Telegram — получишь отчёт прямо сейчас, не дожидаясь ночи

## ✏️ Раздел 5: Как менять промпты через WF05
**Промпт** — это инструкция для ИИ. Например: *"Ты — аналитик. Сделай краткий отчёт по этим данным. Выдели главное."*

### Четыре промпта в системе:

| Название | Для чего | Где используется | Приоритет |
|---|---|---|---|
| `call_summary_prompt` | Саммари одного созвона | WF03, 22:00, сохраняется в open-notebook | ⚡ НАЧНИ С ЭТОГО |
| `chat_summary_prompt` | Саммари Telegram-переписки | WF03, 22:00 | Второестепенно |
| `report_prompt` | Отчёт по команде `/report` | WF04, Telegram-бот | Третье |
| `digest_prompt` | Ежедневный дайджест | WF02, 23:00 | Четвёртое |

> ⚡ **Почему `call_summary_prompt` важнее всего:** Именно он определяет как выглядит саммари каждого созвона которое сохраняется в open-notebook для каждого клиента (часть LEAD-XXXX).

### Как это сделать безопасно:

**Шаг 1:** Открой воркфлоу **WF05 — Prompt Tester** в n8n

**Шаг 2:** Найди ноду **"Set Test Prompt"** (это первый блок в цепочке), нажми двойной клик

**Шаг 3:** Заполни три поля:
- **`save_as`** — напиши точное название промпта (cm. таблицу выше). Например: `call_summary_prompt`
- **`prompt_text`** — вставь новый текст промпта
- **`test_lead_id`** — ID клиента (например `1000023`). Оставь пустым — возьмёт последний транскрипт из БД

**Шаг 4:** Нажми **"Save"** (вверху ноды), потом кнопку **▶ Test workflow** (вверху страницы)

**Шаг 5:** Посмотри ноду **"Show Result"** — там виден результат с новым промптом

**Шаг 6:** Если результат нравится → промпт автоматически сохранился в БД и будет использоваться с этого момента

**Шаг 7:** Если не нравится → просто измени `prompt_text` и повтори. Ничего не сломается.

> 💡 **Подсказка:** Хороший промпт для саммари созвона обычно содержит:
> - Что должно выйти ("Сделай саммари созвона с клиентом")
> - Формат ("Составь список: участники, договорённости, задачи, следующие шаги")
> - Ограничения ("Максимум 500 слов. Только факты, без воды.")

## 🆘 Раздел 6: Что делать если сломалось

### Бот не отвечает на `/report`

**Что проверить:**
1. Открой n8n → Executions → найди последнее выполнение WF04
2. Если красное — нажми на него, посмотри в какой ноде ошибка
3. Частая причина: нода "Get Telegram Updates" не получает сообщения

**Быстрое решение:** Переимпортируй WF04 заново (см. Раздел 3)

---

### Транскрипция не работает (новые файлы не обрабатываются)

**Что проверить:**
1. Открой n8n → Executions → посмотри на WF01
2. Если WF01 вообще не запускается — возможно, он выключен

**Быстрое решение:** Открой WF01 в n8n → убедись что тумблер Active включён

---

### Дайджест не приходит в 23:00

**Что проверить:**
1. Открой n8n → посмотри WF02 → включён ли он (Active)?
2. Зайди в Executions → есть ли запуски WF02?

**Быстрое решение:** Переимпортируй WF02 заново (см. Раздел 3)
