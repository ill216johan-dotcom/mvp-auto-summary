## Что вообще делает эта система?

Коротко: **записи звонков с клиентами → текст → краткие саммари → база знаний по каждому клиенту**.

Как это работает на практике:

1. Запись звонка появляется на сервере (в папке `/mnt/recordings/`)
2. Система автоматически распознаёт речь и превращает аудио в текст
3. ИИ делает краткое резюме: о чём говорили, что обещали, какие дедлайны
4. Саммари сохраняется в базу знаний по конкретному клиенту
5. Каждый день в Telegram приходит дайджест — что было за день
6. Через Dify-чатбот можно задать вопрос: «что обещали клиенту 4405?» — и получить ответ

## Транскрипция: как работает и что выбрать

### Что такое транскрипция?

Это перевод аудио в текст. Без этого шага саммари не появится — ИИ не умеет «слушать» звонок, он читает текст.

Сейчас настроено три варианта. Переключать можно в любой момент, ничего не ломая.

---

### Три варианта — сравнение

#### Вариант 1: Яндекс SpeechKit (сейчас активен)

**Как работает:** запись уходит в Яндекс, там распознаётся, результат возвращается.

**Плюсы:**
- Отличное качество русской речи
- Не нагружает ваш сервер
- Быстро (1–2 минуты на запись)

**Минусы:**
- Платно: **0.60 руб за каждую минуту аудио**
- За что именно платите: за **время записи** (не за количество слов). Тихая 10-минутная запись = 6 рублей, громкая 10-минутная = тоже 6 рублей

**Когда выбирать:** если сервер слабый или нужна максимальная точность и скорость. При небольшом количестве звонков (10–30 в день по 15–30 минут) это ~100–300 руб/день.

**Включить:**
```
STT_PROVIDER=speechkit
```

---

#### Вариант 2: Whisper на своём сервере (бесплатно)

**Как работает:** на сервере запускается локальная ИИ-модель, которая сама распознаёт речь. Ничего никуда не уходит.

**Плюсы:**
- Бесплатно навсегда
- Данные не покидают сервер
- Очень хорошее качество русского (модель medium/large-v3)

**Минусы:**
- Медленно: 10 минут аудио → 5–15 минут обработки (зависит от модели)
- Нужна оперативная память: у вас 7.8 GB, из которых свободно ~2.6 GB → **модель medium (2 GB) встанет, large-v3 (4 GB) — нет**
- При нескольких звонках одновременно — очередь

**Когда выбирать:** когда хотите сэкономить и готовы подождать. Идеально если звонков немного и они не срочные.

**Включить:** (инструкция ниже в разделе «Как переключить провайдер»)

---

#### Вариант 3: AssemblyAI (облако, дешевле SpeechKit)

**Как работает:** как SpeechKit, только другой сервис — американский.

**Плюсы:**
- Дешевле SpeechKit в ~2.5 раза: **~0.23 руб/мин**
- Хорошая поддержка русского языка
- Быстро, не нагружает сервер

**Минусы:**
- Нужен API-ключ (регистрация на assemblyai.com)
- Данные уходят на серверы США

**Когда выбирать:** когда хотите облачный вариант, но дешевле Яндекса.

**Включить:** (инструкция ниже)

Боюсь что данные уйдут за рубеж?

## Что делать если что-то сломалось

### Telegram-бот не отвечает

1. Открыть http://84.252.100.93:5678
2. Найти «04 Telegram Bot Commands» → убедиться что зелёный (Active)
3. Если красный — нажать на тумблер чтобы включить

### Саммари не приходят в Telegram

Возможные причины:
1. **Нет новых записей** — если не было звонков, дайджест не отправляется
2. **Транскрипция не работает** — проверить: `curl http://localhost:9001/health`
3. **Workflow остановился** — проверить в n8n последний запуск WF02 (21:00)

### Файл не транскрибируется

Убедитесь что имя файла начинается с номера клиента:
```
✅ Правильно: 4405_звонок_2026-03-01.ogg
✅ Правильно: 987_meeting.webm
❌ Неправильно: звонок_с_клиентом.ogg  (нет номера в начале)
❌ Неправильно: meeting_987.mp3  (номер не в начале)
```

### Перезапустить все сервисы

Если что-то совсем непонятно — можно перезапустить всё:
```bash
cd /root/mvp-auto-summary
docker compose restart
```

## Технические детали (на случай если всё-таки нужно)

### Где что хранится на сервере

| Что | Где |
|-----|-----|
| Записи звонков | `/mnt/recordings/` |
| Саммари (.md файлы) | `/var/lib/docker/volumes/mvp-auto-summary_summaries_data/` |
| Настройки системы | `/root/mvp-auto-summary/.env` |
| Логи транскрипции | `/tmp/ts.log` (внутри контейнера) |

### Посмотреть логи транскрипции

```bash
docker logs mvp-auto-summary-transcribe-1 --tail=50
```

### Посмотреть что обрабатывалось

```bash
docker exec mvp-auto-summary-postgres-1 psql -U n8n -d n8n \
  -c "SELECT filename, status, lead_id FROM processed_files ORDER BY id DESC LIMIT 10;"
```

### Форматы аудио которые принимает система

- `.ogg` — рекомендуется (самый компактный)
- `.webm` — поддерживается (Jitsi/Jibri пишет в этом формате)
- `.mp3` — поддерживается
- `.wav` — поддерживается

Максимальный размер файла: **100 MB**. Если файл больше — система пропустит его.
